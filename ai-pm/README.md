# AI 产品经理（偏技术）岗位考核

## 1. 考核背景

假设你需要在一个小型团队中，快速验证一个 **AI 对话产品 MVP**。

你的目标不是“做一个完整产品”，而是：

* 用最小成本
* 在最短时间内
* 验证一个 AI 产品是否具备落地可行性

你可以把自己当成：

> **既要做产品决策，也要能亲自下场验证的 AI 产品经理**


## 2. 任务目标

实现一个 **可运行的 AI 对话系统 MVP**，并清楚说明：

* 你为什么这样设计
* 哪些功能你选择不做

技术栈 **不做强制要求**，但：

* 后端建议使用 Python
* 前端可以极简（甚至无前端）


## 3. 功能要求

### ✅ 必须完成（核心）

1. **基础用户体系**

   * 登录 / 注册（可以非常简化）
   * 不要求安全级别达到生产标准

2. **AI 对话能力**

   * 支持文本对话即可
   * 可直接调用大模型 API
   * 或使用 Agent / 框架（非强制）

3. **对话记录持久化**

   * 至少保存：用户、时间、消息内容
   * 可使用任意数据库 / 文件方案

4. **README 说明（非常重要）**
   请在 README 中回答以下问题：

   * 你对这个产品的定位是什么？
   * 你为什么选择当前的技术方案？
   * 当前方案的主要风险或限制是什么？
   * 如果这是一个真实产品，接下来 2 周你会如何迭代？


### ⭐ 可选加分项（非必须）

* 语音输入 / 输出
* Prompt 设计说明
* 多轮上下文控制策略
* Agent 能力设计
* 成本 / Token 使用的简单思考

> **请注意：**
> 可选项不做，不会扣分；
> 盲目堆功能，反而可能减分。


## 4. 我们重点关注什么？

* 你是否像产品经理一样思考，而不仅是“写代码”
* 你是否理解大模型：

  * 能做什么
  * 不适合做什么
* 你是否能清楚表达自己的决策逻辑
* 你如何利用 AI 工具提升效率
